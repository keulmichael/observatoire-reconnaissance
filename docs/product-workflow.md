# Workflow produit

Le parcours cible de l'application est :

```text
Observation
-> Analyse
-> Validation
-> Etude cible
-> Journal permanent
-> Objets scientifiques generes
-> Etats
-> Comparaison
-> Transition
-> Delta
-> Resultats du moteur
-> Tracabilite vers les sources
-> Export / import
```

Une observation validee devient un `ObservationRecord` permanent. Elle reste consultable depuis l'etude, avec ses extraits, les propositions acceptees, modifiees ou rejetees, les objets generes, les moteurs executes et les questions ouvertes.

La page Etude devient le centre du produit. Les vues expertes restent disponibles, mais l'utilisateur peut comprendre une etude sans quitter son espace central.
